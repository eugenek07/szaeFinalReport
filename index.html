Final Report: Wind Tunnel Flow Simulator (WTFS)

Eugene Kang, Shawn Lim, Chin Zong Han, Angeline Lee



Abstract

The Wind Tunnel Flow Simulator project is a modification to the skeleton code for HW4, Clothsim, which modifies the pointmass system and turns them into moving particles. These moving particles interact with each other and with the environment through collisions to accurately move through a closed tunnel system. Collisions and particle speed affect the color of the particles. All in all, the Wind Tunnel Flow Simulator will utilize a 3D environment with simulated air particle collisions to generate a heat map over the surface of different objects.


Technical Approach

Particles

To implement our project, we first modify the parameters within the pointmass struct.

  • Pointmasses are rendered as GL points in a vertical grid
  • Pointmasses move on the horizontal plane: gravity value modifed
  • Spring connections between pointmasses and halfedge mesh rendering of cloth removed
  • Navier-stokes file created
  • Wind particles will collide with object, but initial implementation of Navier-stokes is not accurate

Currently, we have successfully modified the clothsim functionality to render a grid of particles instead of a mesh for the cloth. We modified the .json test files so that the grid is created vertically and changed the gravity value to allow the particles to move horizontally. We've removed the spring classes and their effect on the particles at the moment. To summarize our accomplishments, we've finalized our setup in transitioning the clothsim skeleton code to prepare the implementation of Navier-Stokes.

Particle Based Wind Simulation

Collisions

In order to facilitate air flow, we used Navier-Stokes equations. Objects inside the tunnel and the wind-tunnel walls act as obstacles for the flow.

Results

Following our initial plan of execution, the results show a successful implementation of the wind tunnel. Currently, we believe that progress has been good, with our remaining two weeks, the main thing to implement is Navier Stokes, storage of collision information to affect color, and finally the continuous generation of points per time step to simulate wind. There were a few moments where we had to weigh different options we could take. For instance, we debated on whether to render the pointmasses as GL points or spheres. There were pros and cons to both sides, but for the sake of rendering and the efficiency of the simulation, we decided on proceeding with GL points.
